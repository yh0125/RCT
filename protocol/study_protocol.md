## 1. 背景与立题依据

### 1.1 研究背景

放射检查报告通常使用专业医学术语撰写，面向转诊医师，普通患者难以理解。研究表明放射报告的可读性平均处于高中至大学水平（11-13年级），仅有4%的报告达到8年级以下的患者友好水平。较差的健康素养与癌症患者死亡率增高相关，凸显了有效医学沟通的临床重要性。

大型语言模型（LLM）为弥合这一沟通鸿沟提供了可扩展的解决方案。Prucker等人（Radiology, 2025）在一项200例癌症患者的前瞻性对照试验中证实，LLM简化的肿瘤CT报告显著降低了患者的认知负荷（调整OR=0.18），提升了文本理解度（调整OR=13.28），并将中位阅读时间从7分钟降至2分钟。然而，该研究也发现6%的简化报告存在事实错误，强调了医师审核的必要性。

### 1.2 研究空白

现有研究主要聚焦于文字简化对患者理解的影响，但尚未探索：
1. 在AI文字解读基础上增加**可视化图片解释**是否能进一步提升理解
2. 视觉信息对患者**认知负荷和焦虑**的影响方向（减轻还是加重）
3. 图文结合的解读方式在不同检查类型中的效果差异

### 1.3 理论框架

本研究基于 **双重编码理论（Dual Coding Theory, Paivio 1986）**：信息以语言和视觉双通道呈现时，认知加工更深入、记忆保持更好。同时参考 **认知负荷理论（Cognitive Load Theory, Sweller 1988）**：适当的可视化设计可以降低外在认知负荷，但设计不当的图片也可能增加负荷。

---

## 2. 研究目的

### 2.1 主要目的

评估在AI生成的放射报告文字解读基础上，增加AI生成的可视化解释图片是否能进一步提升患者的文本理解程度、降低认知负荷并改善报告感知。

### 2.2 次要目的

1. 比较两组患者阅读材料前后的担忧/焦虑变化
2. 比较两组的阅读时间
3. 评估不同人口学因素（年龄、学历、基线焦虑）对干预效果的调节作用
4. 评估AI生成内容（文字和图片）的准确性和质量

---

## 3. 研究假设

- **H₁：** 图文解读组患者的文本理解复合评分显著高于纯文字解读组
- **H₂：** 图文解读组患者的认知负荷复合评分显著低于纯文字解读组
- **H₃：** 图文解读组患者的报告感知复合评分显著高于纯文字解读组

## 4. 研究设计

### 4.1 设计类型

前瞻性、单盲、双臂、平行组随机对照试验（RCT）

### 4.2 分组

| 组别 | 代号 | 干预内容 |
|------|------|----------|
| 文字解读组 | A组 | 原始放射报告 + AI生成的通俗文字解读 |
| 图文解读组 | B组 | 原始放射报告 + AI生成的通俗文字解读 + AI生成的可视化解释图片 |

### 4.3 盲法

- **患者（单盲）：** 两组均被告知收到"AI辅助解读材料"，不知晓另一组的材料形式
- **结局评估者：** 数据分析人员对分组情况设盲
- **AI内容质量评审者：** 独立放射科医师对AI生成内容进行盲法质量评审
- **注意：** 由于干预形式差异（有无图片），实施干预的研究人员无法设盲

## 5. 受试者

### 5.1 纳入标准

1. 年龄 ≥ 18 周岁
2. 在研究期间接受了CT、MRI或X光检查并获得正式报告
3. 至少有一次既往同类检查的存档报告（参考 Prucker et al. 的做法）
4. 能够独立阅读中文材料
5. 自愿参加并签署知情同意书

### 5.2 排除标准

1. 无法提供知情同意
2. 认知或身体障碍导致无法完成问卷
3. 语言障碍妨碍研究参与
4. 医疗从业人员（医生、护士、放射技师等）
5. 急诊或病情危重患者
6. 既往参与过本研究

## 6. 干预措施

### 6.1 AI文字解读生成

1. 从PACS/RIS系统获取患者既往放射检查的正式报告
2. 使用LLM（[待定 — 模型及参数]）将专业报告转化为通俗语言
3. 系统提示词设计：参考 Prucker et al. 的固定提示词策略，确保一致性
4. 由至少一名主治及以上放射科医师审核并修正AI生成内容

### 6.2 AI图片生成（仅B组）

1. 基于AI文字解读内容，使用图像生成模型创建解释性示意图
2. 图片内容可包括：解剖位置标注、病灶示意、对比说明等
3. 每份报告配 1-3 张图片，附简要图注
4. 图片需经放射科医师审核：标注准确性、视觉清晰度、是否可能引起不适

### 6.3 质量控制与错误评估

参照 Gertz et al. (Radiology, 2024) 的评估框架，由两名独立放射科医师评估每份AI生成材料：

| 错误类别 | 定义 | 严重程度分级 |
|----------|------|-------------|
| 事实错误 | 原始报告信息被错误改变 | 轻度 / 中度 / 重度 |
| 内容遗漏 | 原始报告的相关信息缺失 | 轻度 / 中度 / 重度 |
| 不当添加 | 不受原始报告支持的新信息 | 轻度 / 中度 / 重度 |
| 图片错误（仅B组） | 图片标注或展示内容与报告不符 | 轻度 / 中度 / 重度 |

评审者间一致性使用加权Cohen's κ（有序变量）和ICC（连续变量）评估。

### 6.4 材料发放

- 所有材料以[待定 — 纸质打印 / 平板电脑]形式提供
- 参与者在安静环境中独立阅读一遍
- 阅读前后分别记录时间

## 7. 结局指标

### 7.1 主要结局指标

参与者在阅读材料后即时填写的 **7点Likert量表**（1=极低/极差，7=极高/极好），分为三个域：

#### 域一：认知负荷（Cognitive Workload）— 改编自NASA-TLX

| 条目 | 评估内容 |
|------|----------|
| CW1 - 心理需求 | 阅读报告并理解其内容所需的脑力劳动程度 |
| CW2 - 挫败感 | 阅读过程中感到不安、沮丧、烦躁、紧张的程度 |
| CW3 - 努力程度 | 阅读报告并理解其内容所需的努力程度 |

#### 域二：文本理解（Text Comprehension）

| 条目 | 评估内容 |
|------|----------|
| TC1 - 清晰度 | 报告提供的信息的清晰程度 |
| TC2 - 可读性 | 报告提供的信息的可读性 |
| TC3 - 可理解性 | 报告提供的信息的可理解程度 |

#### 域三：报告感知（Report Perception）

| 条目 | 评估内容 |
|------|----------|
| RP1 - 有用性 | 报告提供的信息的有用程度 |
| RP2 - 信任度 | 对报告提供的信息的信任程度 |
| RP3 - 信息充分性 | 基于该报告感觉自己被充分告知的程度 |

#### 阅读时间

- 患者自行记录阅读开始和结束时间，精确到分钟

### 7.2 次要结局指标

| 指标 | 评估工具 | 时点 |
|------|----------|------|
| 阅读前担忧程度 | 7点Likert单条目 | 阅读前 |
| 阅读后担忧程度 | 7点Likert单条目 | 阅读后 |
| 视觉辅助评估（仅B组） | 3条目7点Likert | 阅读后 |
| AI内容质量 | 放射科医师5点Likert评分 | 独立评审 |
| 可读性指标 | Flesch-Kincaid等自动化指标 | 文本分析 |
| 患者满意度 | 5条目7点Likert | 阅读后 |
| 开放性反馈 | 自由文本 | 阅读后 |

### 7.3 复合评分计算

参照 Prucker et al. 的方法：
1. 计算各域三个条目的 Cronbach's α
2. 仅当 α ≥ 0.70 时生成复合评分
3. 复合评分 = 各条目z标准化后的均值
4. 对三个域复合评分的比较应用 **Bonferroni校正**（校正后α阈值 = 0.05/3 = 0.0167）

### 7.4 基线数据

- 年龄、性别、最高学历、职业
- 自评健康状况（7点Likert）
- 阅读前担忧/焦虑水平（7点Likert）
- 检查类型、检查部位、是否首次检查
- 检查原因
- 是否已被医生告知结果

## 8. 样本量

### 8.1 计算依据（参考 Prucker et al.）

- 主要结局：7点Likert量表复合评分
- 假设：组间临床意义差异 = 1分（7点量表）
- 标准差：1.5
- 效应量：Cohen's d ≈ 0.67
- 显著性水平：α = 0.05（双侧）
- 检验效能：1-β = 0.90
- 检验方法：比例优势logistic回归

### 8.2 计算结果

每组需 **88** 人。考虑 **10%** 脱落率，每组需招募 **100** 人，总计 **200** 人。

**注意：** 由于本研究两组均接受AI文字解读，组间差异仅在于是否有图片，预期效应量可能小于 Prucker et al.（其对照组为原始报告）。建议进行中期分析以评估效应量假设是否合理。

详细计算见 `analysis/sample_size_calculation.py`。

## 9. 随机化

### 9.1 随机化方法

采用**分层区组随机化（Stratified Block Randomization）**：
- 分层因素：检查类型（CT / MRI / X光）
- 区组大小：4 和 6 随机交替
- 分配比例：1:1

### 9.2 分配隐藏

使用不透明密封信封或中心化电子随机系统。

## 10. 研究流程

```
筛选 → 知情同意 → 基线评估
  ↓
  基线问卷：人口学 + 健康状况 + 阅读前担忧（7点Likert）
  ↓
  记录阅读开始时间
  ↓
随机分组
  ┌──────────────────┬──────────────────┐
  │      A组          │       B组         │
  │   文字解读组      │    图文解读组      │
  ├──────────────────┼──────────────────┤
  │ 原始报告          │ 原始报告          │
  │ + AI文字解读      │ + AI文字解读      │
  │                  │ + AI图片解读      │
  └──────────────────┴──────────────────┘
  ↓
  独立阅读一遍，按正常速度
  ↓
  记录阅读结束时间
  ↓
  阅读后问卷：
    认知负荷（3条目） + 文本理解（3条目） + 报告感知（3条目）
    + 阅读后担忧（1条目） + 满意度（5条目）
    + 视觉辅助评估（B组，3条目） + 开放性问题
  ↓
数据录入 → 统计分析
```

## 11. 统计分析

详见《统计分析计划》（`protocol/statistical_analysis_plan.md`）。

### 11.1 主要分析

- **比例优势logistic回归（Proportional Odds Logistic Regression）**：
  分析各7点Likert条目的组间差异
- **调整变量**：组别、年龄、性别、学历、自评健康状况、基线担忧水平
- 报告 **粗OR** 和 **调整后OR** 及95%CI
- 模型假设使用 **Brant检验** 验证比例优势假设

### 11.2 复合评分分析

- 计算各域 Cronbach's α
- α ≥ 0.70 时由z标准化条目均值生成复合评分
- 复合评分同样使用比例优势logistic回归分析
- 三个域的复合评分比较应用 **Bonferroni校正**（α = 0.0167）

### 11.3 阅读时间

- 使用 **普通最小二乘回归（OLS）** 分析阅读时间的组间差异
- 报告粗β系数和调整后β系数及95%CI

### 11.4 亚组分析

- 按检查类型分层
- 按年龄（以中位数为界）分层
- 按学历水平分层
- 按基线担忧程度分层

### 11.5 敏感性分析

- 意向性治疗（ITT）分析（主要）
- 按方案（Per-Protocol）分析（补充）
